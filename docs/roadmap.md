# lifegame 路线图

中文名：玩中取乐系统。

> 状态：已暂停 / 归档观察（2026-07-10）。以下路线图保留为历史方案，不再代表近期计划。

## Phase 0：纸面系统

目标：先验证信息架构是否真的适合自己。

- 建立人格清单。
- 建立项目清单。
- 每个人格写一句存在目的。
- 每个项目写 `why / status / next_action`。
- 手动录入 10 条思考之地记录。
- 手动提取问题并标记噪音。

完成标准：

- 能从文档看出短期、中期、长期要做什么。
- 能看出最近在问哪些问题。
- 能判断哪些内容不值得进系统。

## Phase 1：本地 Web MVP

目标：做一个能每天打开的 PWA。

- 首页 Today。
- Persona / Project 管理。
- Task 的短中长期视图。
- 思考之地快速录入、自动初判和手动修正。
- 思考记录分流动作：转任务、转判断、转问题、外部影响、噪音、设为项目下一步、已处理。
- Today 总控视图：短期项目、待拍板判断、NPC 输入、最近未处理思考。
- Question Map。
- Agent Registry。
- SQLite 持久化。

完成标准：

- 手机和电脑都能访问。
- 能新增、编辑、过滤、查询。
- 不依赖复杂配置。

## Phase 2：部署到小服务器

目标：稳定自托管。

- Docker Compose。
- SQLite 文件持久化。
- 反向代理和 HTTPS。
- 登录保护。
- 每日备份。
- 移动端 PWA 安装。

完成标准：

- 外网安全访问。
- 数据可备份可迁移。
- 不需要每天维护服务器。

## Phase 3：AI 对话导入

目标：减少手动搬运。

- 支持手动粘贴对话。
- 支持从本地导出的聊天记录导入。
- 自动提取问题、任务、判断候选。
- 用户确认或修正后入库。
- 过滤测试问题和纯工具验证。

完成标准：

- AI 只做候选提取，不直接污染主库。
- 用户可以一键接受 / 丢弃 / 合并。

## Phase 4：多 agent 协作

目标：把不同 agent 当成项目成员管理。

- Agent 职责卡片。
- Agent 边界和交接规则。
- 项目下的 agent 交互记录。
- Handoff 摘要。
- Agent 之间的冲突和重复建议合并。

完成标准：

- 能知道哪个 agent 管什么。
- 能知道 agent 最近给了什么建议。
- 总控可以拍板，不被多 agent 噪音淹没。
