# lifegame 产品简报

中文名：玩中取乐系统。

## 一句话

一个以人格和项目为核心的个人生活总控台，用来愉快地管理目标、任务、思考之地、问题模式和多 agent 协作。

## 不是做什么

- 不是 Plane/Jira：不需要 issue、sprint、workflow、权限、团队协作。
- 不是 Notion 多维表：不要靠表格视图堆字段。
- 不是聊天记录备份器：原始对话不是核心，提炼后的问题和判断才是核心。
- 不是万能人生系统：先服务 `Vault-Life` 的项目和身份。

## 用户画像

只有一个主用户：gyq。

用户同时扮演多种身份，频繁和 AI 交互，希望从对话中看见自己的真实问题、情绪、判断和行动线索。

## 核心体验

打开首页后，应该像进入自己的游戏主界面：清楚、轻松、有掌控感。

- 今天最重要的 3-5 件事。
- 本周需要推进的项目。
- 当前人格状态：谁在前台，谁被忽略了。
- 最近在思考之地里提炼出的真实问题。
- 需要总控拍板的判断。

这个系统的情绪底色不是“你又落后了”，而是“来，看看今天怎么玩得更顺一点”。

## 体验原则

- 愉快优先：不要让系统本身制造负担。
- 提效优先：只呈现能帮下一步行动更清楚的信息。
- 游戏感优先：人格、任务、项目都像人生游戏里的角色和关卡。
- 反思有边界：允许记录情绪，但不把自己困在情绪里。
- 总控拍板：多 agent 可以给建议，但最终回到本人判断。

## 信息架构

### 总控层

总控层处理跨身份的问题：

- 人生原则
- 长期方向
- 近期节奏
- 情绪和能量
- 风险边界
- 复盘

### 人格层

每个人格有自己的：

- 存在目的
- 适用场景
- 做事原则
- 当前状态
- 短中长期任务
- 关联项目
- 关联 agent
- 最近对话和问题

### 项目层

每个项目记录：

- 为什么做
- 现在到哪了
- 下一步做什么
- 卡点是什么
- 属于哪个人格
- 哪些对话和判断影响了它

### 思考之地

思考之地承接和 AI 对话、外部输入、临时想法有关的原话。它不负责执行，只负责分类和记录。

内容进入系统后，不默认保留全文，而是提炼成：

- 思考摘要
- 用户提出的问题
- AI 给出的建议
- 用户真实关切
- 产生的任务
- 产生的判断
- 是否过滤

### 问题层

系统要能回答：

- 我最近总在问什么？
- 哪些问题属于行动问题？
- 哪些问题是情绪问题？
- 哪些问题是测试、闲聊或噪音？
- 哪些问题反复出现但一直没行动？

### NPC 层

外部人员是 NPC：他们会影响判断，但不拥有方向盘。

系统要记录：

- 谁给了输入。
- 能从“XXX 跟我说...”这类文本里自动识别候选 NPC。
- 这个 NPC 对哪个人格 / 项目产生影响。
- 影响权重是高、中、低。
- 这次输入强化、挑战还是改变了某条原则。
- 总控最终是否采纳。

## 首版必须有

1. 人格列表和人格详情。
2. 项目列表和项目详情。
3. 短期 / 中期 / 长期任务视图。
4. 思考之地快速录入。
5. 系统自动初判类型，用户可以手动修正。
6. Agent 列表：每个 agent 服务哪个人格 / 项目。
7. 首页 Today 视图。
8. 思考记录分流动作：把输入推回任务、判断、问题、外部影响或项目下一步。

## 首版暂不做

- 多人协作。
- 完整权限系统。
- 自动抓取所有 AI 平台聊天记录。
- 复杂甘特图和 sprint。
- 内置大模型调用编排。
- 移动端原生 App。

## 成功标准

首版成功不是“功能多”，而是：

- 每天愿意打开。
- 能在 30 秒内知道最近该干什么。
- 能看到自己最近在问什么问题。
- 能把 AI 对话和临时想法变成任务、判断、问题或外部影响记录。
- 不会变成维护负担。
