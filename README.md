# lifegame

中文名：玩中取乐系统。

> 状态：已暂停 / 归档观察（2026-07-10）。当前不作为近期推进项目，保留原型和部署记录，后续恢复前先重新判断是否仍需要独立系统。

`lifegame` 是我们的代号。它是一个帮人生提效、但必须做得愉快的个人生活操作台。

它不是 Plane、Jira、Notion 多维表，也不是单纯待办清单。它的定位是：以 `Vault-Life` 为总控，把不同身份、项目、AI 对话、问题、任务和判断沉淀到一个每天愿意打开的玩中取乐系统。

## 核心隐喻

`Vault-Life` 是人生游戏的总 CEO。

下面挂不同人格 / 身份：

- 总控本人：原则、大方向、反思、情绪管理、人生节奏。
- 元气工作人格：工作里的牛马视角、短期交付、压力管理。
- baiheverse 副业投资人：副业、产品、商业化、合作。
- 个人投资者：仓位、标的、交易判断、复盘。
- 装修经理：装修项目推进、供应商、预算、待确认事项。
- 生活安排者：旅行、家庭、健康、日常安排。

这些人格不是割裂的人，而是同一个人处理不同战场时的工作台。

## 最小目标

每天打开平台时，只回答四个问题：

1. 我短期、中期、长期分别要做什么？
2. 我最近和 AI 聊了什么，哪些值得沉淀？
3. 我一直在问哪些问题，哪些是噪音或测试问题？
4. 每个项目 / 人格现在为什么存在，下一步是什么？

## 气质原则

- 不做复杂项目管理。
- 不做多维表式数据库 UI。
- 不追求全自动，一开始允许半自动导入。
- 先帮人看清楚自己，再帮人管理任务。
- 任务必须挂到人格或项目，否则就是噪音。
- 思考之地不是聊天记录仓库，而是分类和记录入口。
- 使用体验要轻、清楚、愉快，像进入自己的游戏主界面。
- 提效不是压榨自己，而是减少混乱、拖延和无意义切换。

## 首版页面

- `Today`：今天 / 本周最该看的事情。
- `Horizons`：短期、中期、长期目标。
- `Personas`：不同人格的状态、原则、任务、最近对话。
- `Projects`：项目目的、进展、卡点、下一步。
- `思考之地`：承接和 AI 对话产生的思考、问题、外部影响和待分类记录。
- `Question Map`：我反复在问什么。
- `Agents`：不同项目挂的 AI agent、职责和交互记录。

## 推荐技术方向

首版建议做成单人自部署 Web App：

- 前端：Next.js 或 SvelteKit，做成 PWA，电脑和手机都能访问。
- 后端：同框架 API routes，先不拆微服务。
- 数据库：SQLite。
- 部署：一台小服务器 + Docker Compose。
- 备份：数据库每日导出到 vault 或对象存储。

后续再考虑接入 Codex / 飞书 / 各类 AI 会话导出。

## 初始配置

- `config/personas.yaml`：首批人格 / 身份。
- `config/projects.yaml`：首批项目。

这两份文件是 Phase 0 的纸面系统，也是后续 Web MVP 的 seed data。

思考之地采用“先收进来，再判断”的方式：用户不需要先分类，系统会先给出类型初判，之后可以手动修正。它只负责分类和记录，具体执行回到对应人格和项目。

当前原型先用前端脚本做稳定分流：转任务、转判断、转问题、标记外部影响、标记噪音、设为项目下一步、标记已处理。后续接入 AI prompt 后，再把脚本规则升级成更细的语义判断。
